bind_action(action_type, action_controls, action_cb)

Bind action callback to user controls. This allows to set and define an additional action to an event and set the conditions to when and how the event happens.
Parameters:
Name Type Description
action_type number Type of action
action_controls Array Array of sensor types
action_cb function Function which applies logic
Source:
  • addons/fps.js, line 694
Example
var m_ctl = require("controls");
var m_fps = require("fps");
var m_input = require("input");

var action_cb = function(value) {
    console.log(value);
};

// bind custom callback to be executed after pressing the W key or one of the gamepad 
// buttons or after clicking/touching the html element with the "forward_button_id" id
m_fps.bind_action(m_fps.AT_PRESSED, [m_ctl.KEY_W, m_input.GMPD_BUTTON_12,
         "forward_button_id"], action_cb);

Enable FPS controls. This sets keyboard, gamepad and mouse controls. VR is also plug-and-play ready.

Name Type Attributes Default Description

 {} Initialization options.

 The result of the get_canvas() method call HTML element to add event listeners to.

 null Motion callback function

 0 Connected gamepad ID.

 [KEY_W, GMPD_BUTTON_12] Array of sensor types used for foward motion.

jump_sens Array.<number> <optional>
 [KEY_SPACE, GMPD_BUTTON_1] Array of sensor types used for jumping.
fly_sens Array.<number> <optional>
 [KEY_SHIFT, GMPD_BUTTON_7] Array of sensor types used for flying.
rotation_cb CharRotationCallback <optional>
 Callback for camera rotation. If not specified, the default one will be used.
lock_camera boolean <optional>
 false Parent camera to the character
